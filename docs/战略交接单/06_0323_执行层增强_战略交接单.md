# 战略交接单：执行层增强（场景三 V2）

## 业务背景与目标
- 当前场景三已具备“异常上报 + 智能体响应卡片”基础能力。
- 下一轮目标是把“可执行性”做实：让评审看到从异常识别到工单闭环的过程状态，而不只是静态卡片。

## 本轮战略诉求
1. 保持现有飞书风格 UI 不变形，延续单文件可演示模式。
2. 强化“可信数据空间”叙事：每一步都体现最小权限、自动校验、可追踪。
3. 演示节奏控制在 3 分钟内，形成明确 Aha Moment：  
   一线呼救 -> 智能体分级 -> 自动协同 -> 超时升级。

## 演示剧本硬性要求（文案需一致）
- 卡片中新增“协同工单状态轨迹”：
  - 已创建（11:18）
  - 已派发（11:19）
  - 待建设方确认（倒计时 01:59:xx）
- 点击“查看工单详情”后，右侧弹出详情抽屉（纯前端模拟）。
- 抽屉中必须包含三段信息：
  - 工单编号、责任人、当前 SLA 倒计时
  - 可信数据空间校验记录（自动脱敏通过/拒绝项）
  - 升级规则（超时自动抄送二级协同层）

## 交互硬性要求
- 不得引入后端请求；使用前端状态模拟即可。
- 切换群组、打开抽屉、关闭抽屉都需平滑过渡。
- “转人工提报 / 补充现场日志”按钮显示组长权限态（可见但强调角色限制）。

## 验收标准（CR/QA）
- 工单详情抽屉可稳定打开与关闭，不遮挡主消息流关键内容。
- 状态轨迹与倒计时文案展示完整，且不与现有内容冲突。
- 维持当前单文件可运行特性，浏览器直接打开即可演示。

## 优先级
- P0（下次汇报前必须具备）。
