# 大型项目跨组织智能体协作平台 - 资产索引

## 1. 当前可运行原型
- `src/prototypes/im-main-v0.1.html`：IM 内嵌原型基线版本（包含场景一/二/三）。

## 2. 文档导航
- `docs/00_0311_五角色AI研发体系_工作规则.md`：五角色协作总纲。
- `docs/changelog/`：迭代日志与每次变更记录。
- `docs/decisions/`：关键设计决策记录（ADR 风格）。
- `docs/战略交接单/`：产品总监输出战略交接单。
- `docs/工单/`：TPM 派工单与 Bug 修复清单。
- `docs/汇报与演示/`：售前 Demo 剧本与答辩话术。

## 3. 目录归档规则
- 代码与原型放在 `src/`。
- 需求、评审结论、决策放在 `docs/`。
- 历史草稿、过期截图、旧版本放在 `archives/`。

## 4. 命名规范
- 原型文件：`模块-场景-版本.html`，例如 `im-execution-layer-v0.3.html`。
- 文档文件：`[两位流水号]_[MMDD]_[场景主题]_[角色后缀].md`，例如 `04_0323_场景三Demo_售前剧本.md`。
- 资源文件：`业务域_用途_版本.扩展名`，例如 `agent_card_status_v2.png`。

## 5. 每次迭代最小清单
1. 原型变更后，更新 `docs/changelog/` 一条日志。
2. TPM 输出派工单后，前端仅按 `@docs/工单/*.md` 实施。
3. 存在关键取舍时，新增 `docs/decisions/` 一条决策记录。
4. 需要对外演示时，新增一条 `docs/汇报与演示/` 剧本并关联对应场景版本。
