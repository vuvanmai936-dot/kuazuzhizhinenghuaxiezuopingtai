# 前端开发派工单：执行层增强（场景三 V2）

## 输入文件
- `@docs/战略交接单/06_0323_执行层增强_战略交接单.md`

## 开发目标
- 在现有执行层场景基础上，新增“工单状态轨迹 + 工单详情抽屉”，完成从卡片到细节闭环的演示链路。

## 开发范围
1. **工单状态轨迹**
   - 在执行辅助智能体响应卡片内新增状态时间线：
     - 已创建（11:18）
     - 已派发（11:19）
     - 待建设方确认（倒计时 01:59:xx）
2. **工单详情抽屉**
   - 点击“查看工单详情”打开右侧抽屉。
   - 抽屉信息模块：
     - 工单基础信息（编号、责任人、SLA 倒计时）
     - 可信数据空间校验记录（脱敏校验通过/拦截项）
     - 超时升级规则说明
   - 提供关闭按钮，支持点击遮罩关闭。
3. **权限态样式**
   - “转人工提报 / 补充现场日志”按钮增加“组长权限”视觉提示（例如角标或说明字样）。

## 非功能要求
- 仍保持单文件：HTML + Tailwind + 原生 JS。
- 不引入外部本地依赖，不添加后端接口。
- 动画平滑：群切换、抽屉开关都使用 Fade/Slide 过渡。

## QA 验收标准
- 点击“查看工单详情”可稳定开关抽屉，连续操作 5 次无异常。
- 抽屉不遮挡顶部标题与核心消息可读性。
- 执行层与总控层切换后，状态轨迹与按钮逻辑保持正确。
- 浏览器直接打开文件可运行，无控制台报错。

## 交付格式
- 回复需包含：
  - 改动文件路径
  - 实现摘要（3-5条）
  - 动画说明（切换与抽屉）
