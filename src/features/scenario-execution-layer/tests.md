# 执行层场景测试用例（手动回归）

## 范围
- 页面入口：`src/pages/im-main.html`
- 场景：执行层切换、工单抽屉、SLA 倒计时、场景二触发链路

## 用例清单

### CASE-01 会话切换到执行层
- 前置：打开页面，默认在总控群组
- 步骤：
  1. 点击 `子项目A-集成协同群组`
- 预期：
  - Header 标题更新为执行层
  - 状态标签更新为“一线作业与异常上报节点”
  - 执行层卡片渲染完成

### CASE-02 打开与关闭工单详情抽屉
- 前置：已进入执行层
- 步骤：
  1. 点击 `查看工单详情`
  2. 点击遮罩关闭
  3. 再次打开，点击右上角关闭按钮
- 预期：
  - 抽屉可重复开关，无卡死
  - 关闭动作稳定生效

### CASE-03 SLA 倒计时同步
- 前置：执行层卡片已显示
- 步骤：
  1. 打开工单抽屉
  2. 观察卡片与抽屉中的倒计时 5 秒
- 预期：
  - 两处倒计时每秒递减
  - 数值变化一致

### CASE-04 场景二链路可触发
- 前置：总控群组
- 步骤：
  1. 点击 `一键穿透归因`
- 预期：
  - 先出现 loading 提示
  - 再出现归因结果卡片

### CASE-05 来回切换稳定性
- 前置：页面加载完成
- 步骤：
  1. 总控 <-> 执行层 来回切换 5 次
- 预期：
  - 无报错
  - 无明显 UI 错位
  - 按钮与交互保持可用
